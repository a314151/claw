# AGENTS.md

本文件用于定义本仓库的协作偏好与执行约定。
对任何 AI 助手（含代码生成、审查、重构、自动修复）要求如下：

## 0. 最高优先级

1. 进入仓库后，先阅读本文件，再进行分析与修改。
2. 若本文件规则与默认行为冲突，以本文件为准。
3. 未明确事项可自行决策，但必须保持可读性、鲁棒性与安全性。

## 1. 语言与沟通偏好

1. 输出语言：中文。
2. 文档语言：中文优先。
3. 回答风格：简洁、可执行、避免空话。

## 2. 技术栈偏好

1. 主语言：Python。
2. 虚拟环境与依赖管理：统一使用 uv。
3. 默认模型提供商：DeepSeek。
4. 项目方向：LLM + MCP 个人助手，支持本地 Web 可视化运行。

## 3. 代码风格偏好

1. 模块化设计：配置层、模型层、MCP 层、编排层、接口层分离。
2. 保持精简：避免过度抽象和无意义封装。
3. 可读性优先：命名清晰，函数职责单一。
4. 可迭代性：保留扩展点，避免硬编码。

## 4. 质量与安全要求

1. 注重鲁棒性：边界校验、超时控制、错误处理必须具备。
2. 禁止引入明显漏洞：尤其是命令注入、XSS、敏感信息泄露。
3. 变更前后应可运行，必要时提供最小验证步骤。
4. 不在代码、日志或文档中硬编码真实密钥。

## 5. 文档要求

1. README.md 保持中文。
2. 任何新增功能都应同步更新 README 的运行方式与配置说明。
3. 命令示例优先给出 Windows 可直接执行版本。

## 5.1 新增功能约定（2026-03）

1. Web 端默认启用邮箱验证码登录流程，未登录不允许发起聊天请求。
2. 用户历史、画像、MCP 用户配置优先使用 Cloudflare R2 持久化。
3. 不在代码中硬编码 Cloudflare Token、Account ID、邮箱密码等敏感信息。
4. 用户画像是简要摘要：总量不超过 5000 词，单次写入不超过 200 词。
5. 画像写入时机：用户明确要求，或对话轮数每 10 轮触发一次。
6. 每次调用 LLM 时，将用户画像作为系统上下文的一部分注入。
7. 支持 Skill 模块：每个 Skill 需有“总结性题目（标题）+ 关键词 + 执行指令”，调用工具前先做匹配。
8. 用户行为习惯（如活跃时段、工具调用频次、模型使用偏好）应可持久化并可视化编辑。
9. 用户可在头像菜单查看并编辑持久化内容（画像、习惯、Skills、价格表），每次修改自动创建存档并支持回滚。

## 5.2 前端稳定性约定（2026-03）

1. 新增前端依赖接口（如 `/api/pricing`、`/api/user/memory`、`/api/export/answer`）时，必须提供旧版服务兜底提示，避免直接报错中断。
2. 导出按钮默认使用紧凑折叠样式，避免占用过多消息区域。
3. 回答区必须可见 Token 信息；若上游未返回 usage，应明确显示“未返回”，不得静默缺失。
4. 每次提交前至少执行一次“登录 -> 账号菜单按钮 -> 设置/历史/资料库 -> 发送/导出”的按钮巡检并记录结果。

## 6. 提交与改动约定

1. 仅做与目标相关的最小改动。
2. 不随意重排无关代码风格。
3. 不回退用户已有改动，除非用户明确要求。

## 7. 冲突处理

1. 当用户最新指令与本文件冲突时，以“用户最新明确指令”为最高优先。
2. 若存在安全风险，应先提示风险，再给出可行替代方案。
